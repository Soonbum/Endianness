# 엔디안 (Endianness)

* 엔디안이란 무엇인가?
  - 컴퓨터 메모리에 여러 바이트로 이루어진 데이터를 배열하고 저장하는 순서를 의미함

| 리틀 엔디안(Little-Endian) | 빅 엔디안(Big-Endian) |
| -------------------------- | --------------------- |
| 맨 뒤부터 1바이트씩 채워넣음 | 앞부터 순서대로 1바이트씩 넣음 |
| 산술 연산, CPU 데이터 처리가 효율적임 | 인간에게 직관적이고 디버깅이 용이함 |
| 대부분의 CPU 아키텍처에서 사용하는 방식 | 네트워크 통신에서 표준으로 사용하며, 일부 CPU 아키텍처(Motorola 68000, IBM PowerPC/POWER, MIPS)에서도 사용함 |

* 값을 채우는 방식: 4바이트 값(0x12345678)을 메모리에 넣는다고 하면 다음과 같다. (※ MSB: Most Significant Byte, LSB: Least Significant Byte -- 비트가 아니고 바이트임)
  - 리틀 엔디안의 경우
    | [3] MSB | [2] | [1] | [0] LSB |
    | ------- | --- | --- | ------- |
    | 78      | 56  | 34  | 12      |
  - 빅 엔디안의 경우
    | [3] MSB | [2] | [1] | [0] LSB |
    | ------- | --- | --- | ------- |
    | 12      | 34  | 56  | 78      |

* 네트워크 장비에서 빅 엔디안 방식을 채택한 이유
  - 네트워크 스트림은 순차적으로 들어옴
  - 라우터(Router)가 패킷을 수신할 때 목적지 IP 주소의 앞부분을 먼저 읽고 패킷 수신을 완료하기 전에 미리 스위칭을 시작하여 네트워크 지연 시간을 최소화할 수 있음
  - 또한, 개발자가 디버깅할 때에도 빅 엔디안으로 표현하면 해석하기 용이함

* 리틀 엔디안의 장점
  - 초기 인텔(Intel)에서 CPU 하드웨어 회로 설계의 효율성 때문에 이 방식을 채택하였음
  - 산술 연산(덧셈, 곱셈)이 빠르고 단순함
    * 산술 연산의 경우 가장 낮은 자리부터 더해서 올림수(Carry)가 발생하는 식으로 진행함
    * 빅 엔디안이었다면 가장 작은 자릿수를 찾아야 하므로 메모리의 끝부분까지 포인터를 이동시키고, 앞으로 이동하면서 계산을 해야 한다.
    * 리틀 엔디안의 경우 처음 바이트를 꺼내서 바로 더하고, 자릿수가 올라가면서 올림수를 기억한 상태로 다음 주소로 이동해서 연산하면 된다.
  - 데이터 크기 변환(Casting/Downcasting) 오버헤드가 없음
    * 다운캐스팅의 예시로는 4바이트 정수(int)를 1바이트 문자(char)나 2바이트 정수(short)로 변환하는 경우가 있다.
    * 리틀 엔디안의 경우 포인터를 이동시킬 필요 없이, 읽는 바이트 길이만 줄이면 됨
    * 빅 엔디안이었다면 메모리 주소 계산(offset)을 해서 포인터를 이동한 후에 읽어야 함

# 요약


* 빅 엔디안: 사람이 눈으로 읽기 편하고, 네트워크 라우터가 길(IP 주소)을 찾기 편하게 설계된 '네트워크 및 인간 친화적' 방식.

* 리틀 엔디안: CPU가 산술 연산을 하거나 데이터를 메모리에서 떼어낼 때, 주소 계산 없이 가장 빠르고 싸게 동작하도록 설계된 '기계(CPU) 친화적' 방식.

* 이러한 하드웨어적인 이점 덕분에 인텔(x86)과 ARM이라는 두 거대 CPU 제조사가 리틀 엔디안을 채택했고, 오늘날 네트워크 선로(인터넷) 위를 날아다닐 때를 제외하고, 기계 내부에서 일어나는 거의 모든 데이터 처리는 리틀 엔디안으로 통일이 되었습니다.
